1.0.0
-----------------------
Moving immersed boundaries are now supported in 2D!
- IB2d import of dynamic boundary meshes
- all 2D plotting routines updated to handle moving immersed boundaries
- both sticky and frictionless immersed boundary interactions included!
- reworked apply_internal_static_BC for 2D and 3D cases, important bug fixes
- swarm.ib_collision is now ib_collision_idx. It is -1 if the corresponding agent did not encounter a mesh elment during the previous timestep, 
and otherwise gives the index of the first mesh element encountered.
- Swarm and Environment classes are now capitalized.
- Cleans up Swarm and Environment by kicking static methods and loaders to external module. Adds leading underscores to make API cleaner.
- Changes the name of get_positions to apply_agent_model, removes params argument.
- Updated installation instructions and added workflow.
- Optional parallelization of immersed-boundary collision detection: attach any worker pool exposing .map (multiprocessing.Pool, ProcessPoolExecutor, or ThreadPoolExecutor) to a Swarm via the new pool attribute/kwarg. Defaults to serial (pool=None), behavior unchanged.
- get_2D_vorticity vectorized via np.gradient: identical to the old loop on uniform grids, more accurate on non-uniform grids.
- New single-file tutorial for sticky moving boundaries.
- Bug fixes: corrected ib_collision_idx for some collisions; fixed flow_points axis ordering for non-square grids passed directly to Environment(); repaired pytest collection and updated stale intersection/vorticity tests.
- Bug fix: sticky moving-boundary collisions no longer return NaN for perfectly axis-aligned (vertical/horizontal) mesh elements
- Bug fix: motion.highRe_massive_drift now works in 2D. The drag term now broadcasts over the environment's dimension (output unchanged in 3D).
- Bug fix (3D immersed boundaries): agents sliding on a triangle mesh no longer stick at shared triangle edges; this affected coplanar tiled (flat) surfaces and concave folds. 
- Backward-time FTLE is now correct and works; tracer particles only.

Geometry module added to collect workhorse mathematical functions. Explanation of the mathematics and implementation has been added and cleaned up.

Better code organization, including consolidation of IB2d mesh loading

Critical bug fixes in agent-mesh interaction; extensive testing.

Test suite overhaul. The automated tests were reorganized into focused, fast,
deterministic modules under tests/ (geometry, static/moving collisions, flow
generation, temporal interpolation, agent models, swarm lifecycle, analysis, and
I/O loaders). The default `pytest` run is ~1s; the slower full-simulation
parallelization checks are gated behind `--runslow`. Exploratory/visual scripts
moved to tests/manual/ (not collected). Tiny committed fixtures in
tests/fixtures/ (regenerable via tests/fixtures/_gen_fixtures.py) give the first
automated coverage of the moving immersed-boundary import. The new tests
uncovered four latent bugs; all four are now fixed with regression tests (the
sticky moving-boundary NaN, the closest_dist zero-length crash, the
save_fluid/save_2D_vorticity pyvista breakage, and backward-time FTLE, all listed
above), so the suite has no remaining xfails. A few non-blocking follow-ups remain,
e.g. full moving-immersed-boundary support in FTLE.


0.7.0
-----------------------
Coloring for individual agents
Agents can now be individually colored by setting a property. They can also be named. This is documented in the Swarm constructor.

Plotting of agent headings
The direction individual agents are facing/moving can now be plotted in 2D. This is turned on by default. The heading is drawn from the agent's current velocity unless individual heading angles are a specified as an agent property.

Property history
Agent property history can now be optionally saved. This is necessary for plotting agents that change color over time (e.g., highlighting certain agents that fulfill a given condition) and other situations.

New method after_move
This method is called after agents finish moving. It can used to act on their final positions and velocities which was formerly not possible because interactions with immersed boundaries would come after the user defined behavior.

Euler_brownian_motion is now robust to arbitrary starting positions/velocities, not just the current state of all agents together.

define_pic_grid
This new method of the Environment class implements a particle-in-cell method for agents interacting with neighbors. It can be utilized to make user-defined behavior more efficient.

ex_poisson_search.py
New example demonstrating how to handle arbitrary switching times in Planktos and demonstrating a basic intermittent search strategy.

Bug fixes.
